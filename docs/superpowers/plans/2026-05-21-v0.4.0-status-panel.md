# v0.4.0 — Installer status panel: library status surface

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement the installer's slice of the cross-repo `installer-status-panel` contract: a new "Library" panel in the popover that renders GUI-pushed status (handle, total saves, retention breakdown, hydration progress, last activity) by polling the helper's `GET /status` endpoint while the popover is visible.

**Architecture:** One new pure-Python module (`status`) carries the data layer (fetch + parse + format) with TDD coverage. The popover gains a third NSViewController for the Library panel; the Default panel gains a "Library →" link mirroring the existing "More →" link. The status fetch is co-fetched on the existing tray health-poll 5s timer when the popover is visible (R10 in the cross-repo resolved archive); no second timer.

**Tech Stack:** Python 3.11+, PyObjC, pystray, httpx, pytest. macOS Sequoia/Tahoe target. Continues v0.3.0's foundation.

**Spec:** `docs/superpowers/specs/2026-05-21-installer-status-panel.md`

**Cross-repo contract:** [`installer-status-panel.md`](https://github.com/tenorune/bsky-saves-coordination/blob/main/docs/installer-status-panel.md) on `tenorune/bsky-saves-coordination`. Pinned to schema_version 1.

**Coordination prerequisite:** the helper must ship `GET /status` and the GUI must ship the push call before the installer's release goes out. Bump `bsky-saves` pin in `pyproject.toml`'s `[tool.briefcase.app.bsky-saves-launcher].requires` to the helper version that includes the endpoints.

```python
"""Status snapshot from the helper's GET /status endpoint.

Wire and payload defined by the cross-repo contract at
tenorune/bsky-saves-coordination:docs/installer-status-panel.md
(§4.2 endpoint, §4.4 payload, §4.5 panel-side surface).

This module owns the data layer: HTTP fetch + tolerant JSON parse +
display-format helpers. The popover side (popover.py) consumes
StatusSnapshot directly and only handles AppKit rendering.
"""

from __future__ import annotations

import datetime as dt
from dataclasses import dataclass, field
from typing import Any

import httpx


STATUS_URL = "http://127.0.0.1:47826/status"
STATUS_TIMEOUT_S = 2.0
STALENESS_THRESHOLD_S = 300  # 5 minutes, locked by R10


@dataclass(frozen=True)
class HydrationProgress:
    completed: int
    total: int


@dataclass(frozen=True)
class LibraryInfo:
    handle: str | None = None
    did: str | None = None
    total_saves: int | None = None
    by_status: dict[str, int] | None = None


@dataclass(frozen=True)
class StorageInfo:
    mode: str | None = None
    session_ttl_seconds: int | None = None
    browser_bytes_estimate: int | None = None


@dataclass(frozen=True)
class ErrorEntry:
    kind: str
    message: str
    count: int


@dataclass(frozen=True)
class LastActivity:
    kind: str | None = None
    started_at: dt.datetime | None = None
    finished_at: dt.datetime | None = None
    added: int = 0
    removed: int = 0
    errors: list[ErrorEntry] = field(default_factory=list)


@dataclass(frozen=True)
class StatusSnapshot:
    schema_version: int | None = None
    updated_at: dt.datetime | None = None
    current_state: str | None = None
    priority: str | None = None
    library: LibraryInfo = field(default_factory=LibraryInfo)
    hydration: dict[str, HydrationProgress] = field(default_factory=dict)
    storage: StorageInfo | None = None
    last_activity: LastActivity | None = None


def fetch_status(*, token: str, base_url: str = "http://127.0.0.1:47826") -> StatusSnapshot | None:
    """GET /status with bearer auth. Returns None on 404 or any error."""
    try:
        resp = httpx.get(
            f"{base_url}/status",
            headers={"Authorization": f"Bearer {token}"},
            timeout=STATUS_TIMEOUT_S,
        )
    except Exception:
        return None
    if resp.status_code != 200:
        return None
    try:
        data = resp.json()
    except Exception:
        return None
    return _parse_snapshot(data)


def _parse_snapshot(data: Any) -> StatusSnapshot | None:
    if not isinstance(data, dict):
        return None
    return StatusSnapshot(
        schema_version=_safe_int(data.get("schema_version")),
        updated_at=_parse_iso(data.get("updated_at")),
        current_state=_safe_str(data.get("current_state")),
        priority=_safe_str(data.get("priority")),
        library=_parse_library(data.get("library")),
        hydration=_parse_hydration(data.get("hydration")),
        storage=_parse_storage(data.get("storage")),
        last_activity=_parse_last_activity(data.get("last_activity")),
    )


# ... (helpers _parse_library, _parse_hydration, _parse_storage, _parse_last_activity,
#      _safe_int, _safe_str, _parse_iso, format_*)
```

Implementation notes:
- Every parse helper returns `None` (or default) on unexpected shape — never raises.
- `_parse_iso` accepts ISO-8601 with `Z` suffix or explicit offset.
- `format_total_saves`: pluralizes ("1 save" / "N saves"), thousands separator via `f"{n:,}"`.
- `format_retention`: builds `["{n} {label}" for n, label in (lost, unsaved) if n > 0]` joined with " · ". Returns `None` if no non-zero values.
- `format_hydration_rows`: iterates in stable order `["articles", "threads", "images"]`, skips missing keys, returns list of `(label, completed, total)` tuples with label-cased label.
- `format_last_activity`: needs a `_relative_time(then, now)` helper that returns "just now" (<60s), "N min ago" (<1h), "N h ago" (<24h), "N days ago" (<7d), then date.
- `format_staleness`: uses the same `_relative_time` but only returns a string when delta ≥ `STALENESS_THRESHOLD_S`.

```python
def _kick_status_fetch(self) -> None:
    """Immediate-on-show fetch. Subsequent updates come from the tray's
    5s health-poll tick."""
    import threading

    from bsky_saves_launcher import status as s
    from bsky_saves_launcher import token as t

    def worker():
        try:
            tok = t.read_pairing_token()
            if not tok:
                self._on_status_fetched(None)
                return
            snap = s.fetch_status(token=tok)
            self._on_status_fetched(snap)
        except Exception:
            self._on_status_fetched(None)

    threading.Thread(target=worker, daemon=True).start()

def _on_status_fetched(self, snapshot) -> None:
    """Marshal back to main thread to update UI."""
    from Foundation import NSOperationQueue

    def apply():
        self.update_library(snapshot)

    NSOperationQueue.mainQueue().addOperationWithBlock_(apply)
```

Off-thread fetch + main-thread UI update is essential — `fetch_status` is blocking and we can't stall the main runloop.

## Open implementation decisions captured in the spec

These are the seven "questions for the plan" the spec listed. Resolved here so the executor doesn't have to re-think them:

1. **Bar gauge widget:** `NSLevelIndicator` with `NSLevelIndicatorStyleContinuousCapacity`. Set rangeMaximum = total, doubleValue = completed. Static (no animation per tick); spinner for in-flight state is separate.
2. **Numeric formatting:** thousands separators via Python's `f"{n:,}"` — locale-naive but matches the doc-language convention. A future localization pass replaces this with `NSNumberFormatter` if non-en locales land.
3. **Errors badge UX:** small `NSBezelStyleRounded` button styled as a badge, with a `setToolTip_` containing the per-error triples. Tooltip-on-hover only — no click-to-expand inline disclosure in v0.4.0.
4. **Test surface:** the new `bsky_saves_launcher.status` module owns all the testable logic (fetch + parse + format). The popover side stays manual-smoke as today.
5. **Schema-version forward compat:** tolerant parsers ignore unknown fields; missing fields → None. No error UI on a higher schema_version. The dataclass-based snapshot model deliberately uses optional / default-empty for every field.
6. **Localization:** strings inline in code. Out of scope for v0.4.0.
7. **Empty-library payload (just signed in, no fetch yet):** confirmed plausible — payload has `library.handle` but `total_saves` is None. `_render_library_panel` handles this: handle renders; total_label is hidden when None; retention + hydration both render empty (no rows) for this state.
